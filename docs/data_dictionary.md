# Data Dictionary

Project: pi-prospective-archive  
Subtrack: C-US  
Status: Pre-freeze data dictionary  
Target LIVE_FREEZE_DATE: 2026-08-03

## Purpose

This document defines the machine-readable data files generated by the C-US prospective archive.

The goal is to make the archive reusable, auditable, and interpretable by future readers without relying on informal explanations.

Each snapshot run contains:

- raw.csv
- computed.csv
- alert.json
- meta.json
- manifest.sha256

The manifest hashes the local snapshot files. Code, SPEC, and environment hashes are recorded in meta.json.

## Snapshot Directory Structure

Snapshots are stored as:

snapshots/C-US/YYYY-MM-DD/RUN_ID/

Example:

snapshots/C-US/2026-07-10/2026-07-10T223110Z_dry_run/

## Snapshot Status Values

snapshot_status may be:

- dry_run: pre-freeze operational test snapshot
- valid: official post-freeze prospective snapshot
- correction: documented correction snapshot

Dry-run snapshots are not prospective evidence. Valid snapshots begin only after LIVE_* constants are frozen.

## raw.csv

raw.csv stores source observations retrieved for the snapshot.

Expected columns:

### snapshot_id

Unique run identifier for the snapshot.

Example:

2026-07-10T223110Z

### subtrack

Archive subtrack.

For this project:

C-US

### provider

Data provider.

Expected primary value:

FRED

### series_id

Source series identifier.

Current C-US primary live series:

- DFF
- DCPF3M
- DTB3
- TOTBKCR

Additional record-only or reproduction series may include:

- DGS3MO
- TEDRATE

### observation_date

Date of the observation.

Format:

YYYY-MM-DD

### realtime_start

Provider real-time vintage start date when available.

May be empty if not provided.

### realtime_end

Provider real-time vintage end date when available.

May be empty if not provided.

### value

Raw observed value as provided by the data source.

### unit

Unit label when available.

May be empty if not provided.

### fetch_status

Fetch status for the observation.

Expected value for successfully retrieved observations:

ok

### fetched_at_utc

UTC timestamp at which the observation was fetched.

## computed.csv

computed.csv stores aligned channel values, normalized channel values, stress values, and alert-related computed fields.

Expected conceptual fields include:

### date

Computation date.

Format:

YYYY-MM-DD

### rho_raw

Raw rho channel.

Definition:

absolute 5-trading-observation change in DFF

### psi_raw

Raw psi channel.

Definition:

absolute 5-trading-observation change in DCPF3M minus DTB3

### omega_raw

Raw omega channel.

Definition:

TOTBKCR level aligned by as-of LOCF

### rho_hat

Normalized rho channel.

Definition:

rho_raw / LIVE_P99["rho"] for valid snapshots  
rho_raw / dry-run P99 for dry-run snapshots

### psi_hat

Normalized psi channel.

Definition:

psi_raw / LIVE_P99["psi"] for valid snapshots  
psi_raw / dry-run P99 for dry-run snapshots

### omega_hat

Normalized omega channel.

Definition:

omega_raw / LIVE_P99["omega"] for valid snapshots  
omega_raw / dry-run P99 for dry-run snapshots

### S

Multiplicative stress value.

Definition:

S = rho_hat * psi_hat * omega_hat

### Sbar_w

Trailing rolling mean of S over 90 trading observations.

This is causal and uses past observations only.

### Pi

Historical cumulative stress integral from the available computed history.

This field may depend on available history start and is not the primary live archive integral.

### Pi_since_freeze

Prospective cumulative stress integral from LIVE_FREEZE_DATE onward.

Before LIVE_FREEZE_DATE, this value is NaN.

### computed_status

Computation status for the row.

Expected values include:

- ok
- unavailable_fill_limit

ok means all required channel values were available after applying as-of fill rules.

unavailable_fill_limit means one or more required channel values were unavailable because the fill limit was exceeded.

### alert_level

Alert state for the row.

Expected values:

- null before LIVE_MU_SIGMA is frozen
- null when no alert threshold is exceeded
- yellow when Sbar_w > LIVE_MU + 2 * LIVE_SIGMA
- red when Sbar_w > LIVE_MU + 3 * LIVE_SIGMA

## alert.json

alert.json stores the latest alert summary for the snapshot.

Expected fields:

### snapshot_id

Unique run identifier for the snapshot.

### subtrack

Archive subtrack.

Expected value:

C-US

### alert_metric

Metric used for alert classification.

Expected value:

Sbar_w

### asof_date

Latest date with usable computed information for alert evaluation.

### alert_level

Latest alert level.

Expected values:

- null
- yellow
- red

In dry_run snapshots before LIVE_MU_SIGMA is frozen, alert_level is null.

## meta.json

meta.json stores audit metadata for a snapshot.

Expected fields:

### snapshot_id

Unique run identifier.

### subtrack

Archive subtrack.

### spec_version

SPEC version used by the snapshot.

### created_at_utc

UTC timestamp when the snapshot was created.

### code_git_commit

Git commit hash of the code used to generate the snapshot.

### p99_used

P99 constants used for normalization.

Dry-run snapshots may use temporary dry-run P99 values.

Valid snapshots must use frozen LIVE_P99.

### mu_sigma_used

Alert threshold constants used for valid snapshots.

Dry-run snapshots before freeze use null.

Valid snapshots must use frozen LIVE_MU_SIGMA.

### freeze_date

Freeze date associated with the run.

For valid snapshots, this must match LIVE_FREEZE_DATE.

### spec_common_sha256

SHA-256 hash of the common SPEC file.

### spec_subtrack_sha256

SHA-256 hash of the C-US subtrack SPEC file.

### environment_hash

SHA-256 hash representing the recorded software environment lock file.

### snapshot_status

Snapshot status.

Expected values:

- dry_run
- valid
- correction

### supersedes

Snapshot or run identifier superseded by a correction snapshot.

Usually null for dry_run and valid snapshots.

### correction_reason

Controlled correction reason when snapshot_status is correction.

Usually null otherwise.

### data_sources

List of source series used in the snapshot.

Expected primary live sources:

- DCPF3M
- DFF
- DTB3
- TOTBKCR

### raw_sha256

SHA-256 hash of raw.csv.

### computed_sha256

SHA-256 hash of computed.csv.

### alert_sha256

SHA-256 hash of alert.json.

## manifest.sha256

manifest.sha256 stores SHA-256 hashes for local snapshot files.

Expected entries:

- raw.csv
- computed.csv
- alert.json
- meta.json

The manifest intentionally covers only local snapshot files.

SPEC, code, and environment integrity are recorded in meta.json through hashes and code_git_commit.

## Missingness and Fill Rules

As-of fill limits:

- DFF: 5 calendar days
- DCPF3M: 5 calendar days
- DTB3: 5 calendar days
- DGS3MO: 5 calendar days
- TEDRATE: 5 calendar days
- TOTBKCR: 14 calendar days

Rows exceeding fill limits are not silently imputed as valid observations.

## No-lookahead Rule

Live computation must not use:

- future observations
- centered rolling windows
- backward fill
- noncausal interpolation

Live alignment uses past observations only.

## Reuse Notes

Users should cite the archive release and preserve snapshot_id, code_git_commit, and manifest hashes when reusing snapshot data.

Dry-run snapshots should not be treated as prospective evidence.

Valid snapshots begin only after the freeze release.
